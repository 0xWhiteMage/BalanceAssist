# Temporary Session Retention

Temporary working drafts expire 24 hours after meaningful authenticated activity. The GitHub Actions expiry worker runs every five minutes on a best-effort basis and deletes expired session rows; database foreign keys delete session-owned rows, including pending outbox records. Before Telegram delivery, the dispatcher authoritatively rechecks that its claimed handoff still belongs to an existing, unexpired session. It suppresses a handoff that no longer qualifies and emits a non-PII `handoff_suppressed` event. Private Storage cleanup is not implemented in this slice, so no storage-deletion claim is made here. Expiry cannot retract messages or files already sent to Telegram or downstream systems.
