# Temporary Session Retention

Temporary working drafts expire 24 hours after meaningful authenticated activity. The GitHub Actions expiry worker runs every five minutes on a best-effort basis. It deletes expired sessions and their unclaimed pending or completed outbox rows, but defers deletion while a dispatcher holds a valid handoff lease. Claiming atomically checks expiry and producer-transfer consent; an expired or revoked unclaimed handoff is suppressed without sending and emits a non-PII `handoff_suppressed` event. Each claim has an ownership token. Before calling Telegram, the dispatcher atomically changes its owned `claiming` row to `sending` and reserves it for 90 seconds. Telegram calls, including response-body parsing, are hard-bounded to 45 seconds, so a normally running sender remains reserved through the call with a 45-second margin; only the current `sending` token can complete or fail it. A `sending` row is retried after its reservation expires. A process pause longer than the 90-second reservation, a process crash, or loss of Telegram's acknowledgement can leave whether Telegram accepted the message unknowable; the retry path is therefore at-least-once and can duplicate an externally accepted send. Expiry and consent revocation suppress only unclaimed handoffs. Private attachment objects are removed by the same worker before their metadata changes to `expired`; failed object deletion leaves metadata stored for a later retry. This is best-effort, so automatic removal is not promised when the cleanup worker or Storage is unavailable. Files are never sent to Telegram in this storage slice.
