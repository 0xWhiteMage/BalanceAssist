# Temporary Session Retention

Temporary working drafts expire 24 hours after meaningful authenticated activity. The GitHub Actions expiry worker runs every five minutes on a best-effort basis. It deletes expired sessions and their unclaimed pending or completed outbox rows, but defers deletion while a dispatcher holds a valid handoff lease. Claiming atomically checks expiry and producer-transfer consent; an expired or revoked unclaimed handoff is suppressed without sending and emits a non-PII `handoff_suppressed` event. Each claim has an ownership token. Completion, failure, and release require that token while the row remains actively claimed, so a stale worker cannot overwrite a newer claim. The dispatcher renews its two-minute lease immediately before a Telegram send and bounds the send to 45 seconds, preventing a known active sender from being reclaimed concurrently. A process crash or remote acknowledgement loss after Telegram accepts a send remains an unavoidable at-least-once ambiguity; no system can retract that external transfer. Once a dispatcher has claimed a handoff, it is authorized to complete that in-flight external transfer: expiry or revocation cannot retract it. Private Storage cleanup is not implemented in this slice, so no storage-deletion claim is made here. Expiry cannot retract messages or files already sent to Telegram or downstream systems.
